# 2.3. Módulo Notação UML – Modelagem Organizacional OU Casos de Uso


## Introdução

A **Modelagem Organizacional na UML (Unified Modeling Language)** descreve a arquitetura funcional do sistema, evidenciando as relações entre os atores e as funcionalidades. 

Essa etapa é essencial para a documentação e análise dos requisitos, pois permite identificar as funcionalidades, ou seja, os serviços oferecidos pelo sistema, além de compreender como eles serão utilizados no contexto organizacional ([LARMAN, 2000](http://www.kufunda.net/publicdocs/Utilizando%20UML%20e%20padr%C3%B5es%20(Craig%20Larman).pdf)).


## Metodologia

No presente projeto, foram documentados os seguintes tipos de diagramas UML que compõem a modelagem organizacional ([SERRANO, 2026](https://unbarqdsw2026-1-turma01.github.io/2026.1-T01-_G4_FCTE_Hoje_Entrega_02/Assets/Referencias/Ref_modelagem_UML_estatica.pdf)):

- [Diagrama de Caso de Uso](/Modelagem/2.3.1.DiagramaDeCasosDeUso.md): O diagrama de caso de uso descreve o comportamento do sistema em interação com atores externos. Ele *representa, de forma clara, como o sistema responde às ações dos usuários, detalhando as funcionalidades oferecidas e os cenários de uso associados*.
- [Diagrama de Pacotes](/Modelagem/2.3.2.DiagramaDePacotes.md): O diagrama de pacotes é uma *representação estrutural que permite organizar o sistema em módulos, oferecendo uma visão hierárquica e organizada de seus elementos*. Ele evidencia a divisão do sistema em pacotes, demonstrando como os diferentes componentes estão agrupados e relacionados.
