# 1.5.3. Lista de Requisitos

## Introdução
Com base nos *slides disponibilizados pela professora Milene Serrano*, a **Lista de Requisitos** é um artefato fundamental da Engenharia de Requisitos que define as funcionalidades (o que o sistema deve fazer) e as restrições (como o sistema deve operar) de um projeto. O objetivo é estabelecer um contrato claro entre os stakeholders e a equipe de desenvolvimento, garantindo que o escopo do sistema **FCTE Hoje** esteja bem definido e testável.

## Metodologia
A construção desta lista foi realizada através da consolidação de dados obtidos via **Brainstorming** (realizado com o grupo de trabalho) e **Questionários** (aplicados aos usuários finais). Após a coleta inicial, os requisitos passaram por um processo de:

1. **Eliminação de Duplicidade:** Cruzamento das fontes para remover itens redundantes.
2. **Refinamento SMART:** Cada requisito foi revisado para garantir que fosse **E**specífico, **M**ensurável, **A**tingível, **R**elevante e **T**emporal.
3. **Classificação:** Separação entre Requisitos Funcionais (RF) e Requisitos Não-Funcionais (RNF).

## Requisitos Funcionais (RF)
Referem-se às ações que o sistema deve ser capaz de executar.

| Identificador | Descrição |
| :--- | :--- |
| **RF01** | Listar notícias acadêmicas da FCTE com atualização em tempo real. |
| **RF02** | Disponibilizar calendário de eventos, palestras e cronograma acadêmico. |
| **RF03** | Listar editais abertos com filtros por categoria (pesquisa, extensão, monitoria). |
| **RF04** | Listar cardápio atualizado do Restaurante Universitário (RU). |
| **RF05** | Centralizar informações de projetos, equipes de competição e esportes (Vida Estudantil). |
| **RF06** | Permitir busca de conteúdos via palavras-chave. |
| **RF07** | Oferecer filtros de notícias por categoria e data. |
| **RF08** | Criar seção personalizada baseada nos temas de interesse do usuário. |
| **RF09** | Integrar com Portal do Aluno para avisos e informações acadêmicas. |
| **RF10** | Permitir exportação de eventos para agendas pessoais (ex: Google Agenda). |
| **RF11** | Configurar notificações push personalizadas por periodicidade e interesse. |
| **RF12** | Gerar resumo diário/semanal das principais informações baseadas nos interesses do usuário. |

## Requisitos Não-Funcionais (RNF)
Referem-se às qualidades e restrições técnicas do sistema.

| Identificador | Descrição |
| :--- | :--- |
| **RNF01** | **Portabilidade:** Interface responsiva adaptada para Android 11+ e iOS 12+. |
| **RNF02** | **Desempenho:** Uso de notificações push nativas para otimização de bateria e performance. |
| **RNF03** | **Disponibilidade:** Suporte a modo offline para consulta de dados em cache do dia. |
| **RNF04** | **Confiabilidade:** Sincronização obrigatória de dados críticos em intervalos de 24h. |
| **RNF05** | **Usabilidade:** Interface baseada na identidade visual oficial da faculdade FCTE. |
